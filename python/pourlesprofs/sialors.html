Structure conditionnelle

Sauf dans le cas de fonctions définies par morceaux (comme par exemple la fonction valeur absolue, la fonction signe, la fonction de Heaviside, la fonction rampe, ...) on va s'éloigner ici un peu des fonctions mathématiques et se placer plutôt dans une structure de disjonction de cas.
En Python, on trouve trois type de structure conditionnelle.
  1. SI une condition est vérifiée ALORS on effectue une ou plusieurs instructions.
    En Python, on commence par le mot clef if suivi de la condition à vérifier. Cette ligne est toujours terminée par deux points :. Ces deux points signifie ALORS. Après ces deux points, comme pour les fonctions, on valide la ligne avec la touche Entrée. Le curseur se place directement à la ligne suivante avec une indentation. Chaque instruction qui doit être exécutée si la condition est respectée doit garder au minimum cette indentation. 
        if condition :
        instruction1
        instruction2
        instruction3
        ...

    Exemple

    Si le client dépense plus de 150 € alors il bénéficie d'une réduction de 5%. 
        if prix > 150 :
        nouveau_prix = prix * (1 - 5/100)
    ou mieux encore 
        if prix >= 150 :
        prix = prix * (1 - 5/100)
    Les conditions sont obtenues en testant deux expressions avec les symboles suivants :
    est-ce vrai que A égale B ? A == B
    est-ce vrai que A est plus grand que B ? A > B
    est-ce vrai que A est plus grand ou égal à B ? A >= B
    est-ce vrai que A est plus petit que B ? A < B
    est-ce vrai que A est plus petit ou égal à B ? A <= B
    est-ce vrai que A n'est pas égal à B ? A != B

  2. SI une condition est vérifiée ALORS on effectue une ou plusieurs instructions SINON on effectue une ou plusieurs autres instructions.
    En Python, on garde le premier bloc avec le if et l'ensemble des instructions à effectuer si la condition est vérifiée. Pour la ligne suivante, on supprime l'indentation pour signifier que l'on va passer à un autre cas. On écrit else : pour SINON. On valide la ligne avec la touche Entrée et on crée visuellement un bloc d'une autre série d'instructions qui doivent être exécutées si la condition n'est pas respectée. Visuellement, on a bien deux blocs d'instructions bien distincts. 
        if condition :
        instruction1
        instruction2
        instruction3
        ...
    else :
        autreIntruction1
        autreIntruction2
        autreIntruction3
        ...

    Exemple

    C'est jour de solde. Tout client bénéficie d'une réduction de 5%. Les clients qui dépensent plus de 150 €, bénéficient d'une réduction exceptionnelle de 10%. 
        if prix > 150 :
        prix = prix * (1 - 10/100)
    else :
        prix = prix * (1 - 5/100)
  3. Lorsque l'on a une disjonction de plus de 2 cas. On utilise alors le mot clef elif (contraction de ELSE IF) suivi d'une nouvelle condition à vérifier si la précédente ne l'a pas été, suivie des deux points : habituels et d'une nouvelle indentation. On peut rajouter autant de blocks elif que nécessaire. Cette série de blocks peut se terminer par un else : pour gérer tous les autres cas non détaillés. 
        if condition :
        instruction1
        instruction2
        instruction3
        ...
    elif nouvelleCondition :
        autreIntruction1
        autreIntruction2
        autreIntruction3
        ...
    else :
        encoreAutreInstruction1
        encoreAutreInstruction2
        encoreAutreInstruction3
        ...

    Exemple

    C'est jour de solde. Tout client bénéficie d'une réduction de 5%. Les clients qui dépensent plus de 150 €, bénéficient d'une réduction de 10% et ceux qui dépensent plus de 250 € d'une réduction de 15%. 
        if prix > 150 :
        prix = prix * (1 - 10/100)
    elif prix > 250 :
        prix = prix * (1 - 15/100)
    else :
        prix = prix * (1 - 5/100)

Problème 2

Ce problème est une suite du problème de la première partie sur les fonctions.

  1. Programmer une fonction qui permet de savoir si un quadrilatere est un trapèze connaissant les coordonnées dans un repère des 4 sommets.
  2. Programmer une fonction qui permet de savoir si un quadrilatere est un parallélogramme connaissant les coordonnées dans un repère des 4 sommets.
  3. Programmer une fonction qui permet de savoir si un quadrilatere est un losange connaissant les coordonnées dans un repère orthonormé des 4 sommets.
  4. Programmer une fonction qui permet de savoir si un quadrilatere est un rectangle connaissant les coordonnées dans un repère orthonormé des 4 sommets.
  5. Programmer une fonction qui permet de savoir si un quadrilatere est un carré connaissant les coordonnées dans un repère orthonormé des 4 sommets.
Lien Basthon pour programmer Lien pour la correction
Aide
Utiliser les fonctions de l'exercice 3 de la première partie sur les fonctions. Pour la fonction distance, on la modifiera légèrement en distanceAuCarre pour donenr la distance au carré, ainsi on n'utilise plus la fonction sqrt qui renvoie des valeurs décimales (en informatique on dit des nombres flottants) et approchées . Lorsque l'on effectue des tests d'égalités il est possible d'obtenir alors des réponses fausses alors qu'il y a bien égalité. Sans rentrer dans les détails, on peut tester : 0.1 + 0.2 == 0.3 !!!
Pour aller plus loin, vous pouvez lire la page sur les nombres flottants.

Conclusion

Dans les problèmes 1 et 2, on peut remarquer à chaque fois que le programme est décomposé en sous-programmes, la fonction finale utilisant une ou plusieurs autres fonctions déjà définies. Cette façon de faire permet de simplifier l'écriture et la lecture d'une fonction principale. Par exemple, si un quadrilatère est un losange et un rectangle alors c'est un carré. De plus, en cas de bogue, comme les fonctions secondaires ont déjà été programmées et testés en amont, l'erreur doit se trouver dans la fonction principale.

La modularité

La modularité est un principe essentiel en programmation. L'idée est de découper son programme en de petits sous-programmes. Autrement dit, éviter de programmer une énorme fonction qui fait plus de 500 lignes, mais plutôt l'écrire avec de ptites fontions qui s'appellent les unes après les autres. Les intérêts sont multiples :

  • le programme sera plus facile à écrire car il sera composé de petits taches simples à écrire puis à vérifier au fur et à mesure
  • s'il y a un bug, il sera plus facile à trouver car l'erreur sera présent dans une petite fonction
  • les sous-programmes écrits peuvent être utiles dans d'autres cas
Par contre cette façon de programme implique d'anticiper en amont le découpage du programme en sous-taches et surtout d'utiliser le mot-clef return à la fin de chaque fonction afin de pouvoir récupérer le résultat obtenu plutôt que la fonction print.

Un mauvais exemple

Avec la fonction print : 
    def f(x):
        print(x**2 - 3*x + 1)
    
    def tauxAccroissement(f,a,b):
        return (f(a) - f(b)) / (a - b)
Dans cet exemple, si on veut calculer les images par f des nombres -2, 4, 3, nous avons l'impression que tout se passe bien. 
    >>> f(-2)
    11
    >>> f(4)
    5
    >>> f(3)
    1
Par contre, si vous taper la commande suivante dans la console, vous obtenez un message d'erreur car la fonction f ne revnoie rien et la fonction tauxAccroissement ne peut effectuer le calcul demandé. 

    >>> tauxAccroissement(f, -2, 4)
    11
    5
    Traceback (most recent call last):
    File "<input>", line 1, in <module>
    File "<input>", line 5, in tauxAccroissement
    TypeError: unsupported operand type(s) for -: 'NoneType' and 'NoneType'>>> tauxAccroissement(f, -2, 4)

Un bon exemple

Avec le mot clef return 
    def f_v2(x):
        return x**2 - 3*x + 1
Pour le calcul du taux d'accroissement, nous n'avons plus de problème. 
    >>> tauxAccroissement(f_v2, -2, 4)
    -1.0

Un dernier très bon exemple

Dans le problème 2 de la partie sur les structures conditionnelles, nous avons vu un très bon exemple de modularité. Connaissant les coordonnées de 4 points dans un repère orthonormé, nous souhaitons savoir la nature du quadrilatère obtenu. Pour y parvenir, j'ai d'abord écrit une fonction qui calcule les coordonnées du milieu de deux points et une autre fonction qui calcule la distance entre deux points. J'utilise ensuite ces deux fonctions dans mes fonctions principales. On peut retrouver le programme sur Basthon en suivant ce lien.
